# Base de Datos

InversiónSegura utiliza **Supabase** (PostgreSQL) como motor de base de datos principal, gestionando el historial de búsquedas y la autenticación de usuarios.

## Esquema Inicial

El siguiente script SQL define la estructura necesaria para el correcto funcionamiento del historial de búsquedas, asegurando unicidad por usuario y símbolo.

```sql
-- 1. Crear tabla historial_busquedas
CREATE TABLE IF NOT EXISTS public.historial_busquedas (
  id BIGINT PRIMARY KEY GENERATED BY DEFAULT AS IDENTITY,
  usuario_id UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  simbolo TEXT NOT NULL,
  buscado_en TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
  -- Restricción UNIQUE para evitar duplicados
  UNIQUE(usuario_id, simbolo)
);

-- 2. Índice para búsquedas rápidas
CREATE INDEX IF NOT EXISTS idx_historial_usuario 
ON public.historial_busquedas(usuario_id, buscado_en DESC);
```
